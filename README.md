# VRML Point Cloud Reduction

大学の画像・映像コンテンツ演習で行った、電子地図データをもとに都市空間をVRMLで表現するグループプロジェクトです。

約1m間隔の格子状標高データを処理し、後楽園キャンパス周辺の3D都市モデルを作成しました。

本リポジトリでは、プロジェクト全体の処理を統合したコードのうち、主に私が担当した **点群データの削減処理** を中心に公開しています。

## Overview

このプロジェクトでは、電子地図データから都市空間の3Dモデルを生成し、VRML形式で可視化することを目的としました。

入力データは、各地点の `x, y, z` 座標を持つ点群データです。  
ただし、元データには以下のような特徴がありました。

- 約1m間隔の格子状標高データである
- 各地点には上空から見たときの最上位点の標高が記録されている
- 地面、建物、道路、樹木などの分類情報は含まれていない
- 一部の地点には欠損値が含まれている

元データをそのままポリゴン化すると点数が多くなり、VRML表示が重くなる可能性があります。  
そのため、都市形状の特徴をできるだけ保ちながら、点群データを削減する必要がありました。

## My Role

私は本プロジェクトの中で、主に **点群データの削減処理** を担当しました。

具体的には、点群を領域ごとに分割し、各領域の高さ変化に応じて削減の強さを変える処理を実装しました。

単純に一定間隔で点を間引くと、平坦な地面だけでなく、建物の角や段差などの重要な形状まで失われる可能性があります。

そこで、各領域の標高差をもとに削減の粒度を変え、平坦な領域では大きく削減し、高さ変化が大きい領域では細かく点を残すようにしました。

なお、欠損点補完、地面判定、VRML出力などはプロジェクト全体の処理としてコード内に統合されていますが、私の主担当はデータ削減部分です。

## My Implementation: Adaptive Point Reduction

私が担当したデータ削減処理では、点群全体を `8 × 8` のタイルに分割し、各タイルの高さ変化をもとに削減の粒度を変えました。

### 1. Tile-based Analysis

まず、点群全体を複数のタイルに分割します。

各タイル内で、標高値 `z` の最大値と最小値の差を計算します。

```python
z_range = max(z_values) - min(z_values)
```

この `z_range` を、その領域の高さ変化の大きさとして利用しました。

`z_range` が大きい領域は、建物や段差などの立体的な特徴を含む可能性が高いと考えられます。  
一方で、`z_range` が小さい領域は、比較的平坦な地面や道路である可能性が高いと考えられます。

### 2. Adaptive Grid Size

各タイルの `z_range` に応じて、削減に用いるグリッドサイズを変化させました。

```python
if z_range > 8.0:
    grid_size = 2.0
elif z_range > 4.0:
    grid_size = 4.0
elif z_range > 2.0:
    grid_size = 10.0
else:
    grid_size = 30.0
```

高さ変化が大きい領域では小さいグリッドを使い、点を細かく残します。

高さ変化が小さい領域では大きいグリッドを使い、点数を大きく削減します。

このように、領域ごとの高さ変化に応じて削減率を変えることで、平坦な領域では効率よく点数を減らし、建物や段差がある領域では形状を残しやすくしました。

### 3. Representative Point Selection

各グリッド内では、含まれる点の高さのばらつきを確認します。

グリッド内の高さ差が小さい場合は、代表点のみを残します。  
一方で、高さ差が大きい場合は、建物の側面や段差が含まれている可能性があるため、点を多めに残すようにしました。

これにより、単純な一律間引きよりも、都市形状の特徴を保ちやすくしています。

## Processing Flow

プロジェクト全体の処理の流れは以下の通りです。

1. 標高データを読み込む
2. 欠損している標高値を補完する
3. 仮地面を推定する
4. 点群を `8 × 8` のタイルに分割する
5. 各タイルの高さ変化を計算する
6. 高さ変化に応じて点群を削減する
7. 削減後の点群に対してDelaunay三角形分割を行う
8. 地面／非地面を色で区別する
9. 1つのVRMLファイルとして出力する

この中で、私が主に担当したのは 4〜6 のデータ削減処理です。

## Result

点群削減処理により、元データの点数を削減し、VRML上でより軽く表示できるようにしました。

特に、平坦な領域では大きく点数を減らし、高さ変化が大きい領域では点を細かく残すことで、データ量の削減と形状保持の両立を目指しました。

ポスター発表時点では、データ量を約70%削減しながら、見た目の品質を大きく損なわない結果を得ました。

## Technical Points

このプロジェクトで工夫した点は、点群を一律に削減するのではなく、領域ごとの高さ変化をもとに削減率を変えた点です。

都市データでは、平坦な道路や地面と、建物の屋上・側面・段差が同じ点群データ内に混在しています。

そのため、全体に同じ削減処理をかけると、平坦な領域では点が残りすぎたり、逆に建物周辺では形状が失われたりする可能性があります。

そこで、各タイルの `z_range` をもとに削減の粒度を変えることで、平坦な領域では大きく削減し、立体的な領域では細かく残すようにしました。

## Reflection

このプロジェクトを通して、3Dデータ処理では「データ量を減らすこと」と「形状の特徴を保つこと」のバランスが重要であることを学びました。

特に、同じ点群データでも、地面・建物・段差などの性質が場所によって異なるため、一律な処理ではなく、局所的な特徴に応じて処理を変える必要があると感じました。

また、VRMLとして出力することで、削減アルゴリズムの結果が実際の見た目にどのように影響するかを確認でき、数値的な削減率だけでなく、視覚的な品質も重要であることを理解しました。
