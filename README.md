# VRML City Model Optimization

大学の画像・映像コンテンツ演習で行った、電子地図データをもとに都市空間をVRMLで再現するグループプロジェクトです。

約1m間隔の格子状標高データを処理し、後楽園キャンパス周辺の都市空間を3Dモデルとして表現しました。

本リポジトリでは、プロジェクト全体の処理のうち、主に私が担当した **点群データの削減処理** を中心に公開しています。

## Overview

このプロジェクトでは、電子地図データから地面・建物などを含む3D都市モデルを生成し、VRML形式で出力することを目的としました。

元データは点数が非常に多く、そのままポリゴン化すると表示処理が重くなるため、建物や地形の特徴をできるだけ保ちながらデータ量を削減する必要がありました。

また、入力データには地面・建物・道路・樹木などの分類情報が含まれていないため、点群データの高さ情報をもとに、地面と地面以外を区別する処理もプロジェクト全体で行いました。

## My Role

私は本プロジェクトの中で、主に **点群データの削減処理** を担当しました。

具体的には、点群を領域ごとに分割し、各領域の高さ変化に応じて削減の強さを変える処理を実装しました。

単純に一定間隔で点を間引くと、建物の角や段差などの重要な形状が失われやすくなります。  
そのため、各領域の標高差をもとに削減率を変えることで、データ量の削減と見た目の品質保持の両立を目指しました。

なお、欠損点補完、地面判定、VRML出力などはプロジェクト全体の処理として統合されていますが、私の主担当はデータ削減部分です。

## Project Goal

本プロジェクトの目標は以下の2点です。

- データ量を削減し、VRML表示を軽量化すること
- 建物や地形の特徴をできるだけ保持すること

授業課題では、処理後のデータをVRML形式で出力し、全体形状・地面形状・地面以外の形状を表現することが求められていました。  
また、データ削減率や原データとの差分を用いた数値評価も評価項目として設定されていました。

## Features

このプロジェクト全体では、以下の処理を行っています。

- 電子地図データの読み込み
- 欠損標高値の補完
- 地面／非地面の判定
- 点群データの削減
- Delaunay三角形分割によるポリゴン生成
- VRML形式での出力

このうち、私が主に担当したのは **点群データの削減処理** です。

## My Implementation

私が担当したデータ削減処理では、点群全体を `8 × 8` のタイルに分割し、各タイルの高さ変化をもとに削減の粒度を変えました。

各タイルでは、標高値の最大値と最小値の差を計算します。

```python
z_range = max(z_values) - min(z_values)
```

`z_range` が大きい領域は、建物や段差などの立体的な特徴を含む可能性が高いため、細かく点を残します。

一方で、`z_range` が小さい領域は比較的平坦な領域である可能性が高いため、大きく点を削減します。

### Adaptive Grid Size

各タイルの高さ変化に応じて、削減に用いるグリッドサイズを変化させました。

```python
if z_range > 8.0:
    grid_size = 2.0
elif z_range > 4.0:
    grid_size = 4.0
elif z_range > 2.0:
    grid_size = 10.0
else:
    grid_size = 30.0
```

高さ変化が大きい領域では小さいグリッドを使い、細かい形状を保持します。

高さ変化が小さい領域では大きいグリッドを使い、点数を大きく削減します。

このように、領域ごとの高さ変化に応じてグリッドサイズを変えることで、平坦な領域では大きく削減し、建物や段差がある領域では形状を残しやすくしました。

### Representative Point Selection

各グリッド内では、標高値のばらつきを確認し、平坦な領域では代表点のみを残しました。

一方で、グリッド内の高さ差が大きい場合は、建物の側面や段差が含まれている可能性があるため、点を多めに残すようにしました。

これにより、単純な平均化や一律な間引きよりも、建物の輪郭や高さ変化を保ちやすくしています。

## Algorithm Overview

プロジェクト全体の処理の流れは以下の通りです。

1. 電子地図データを読み込む
2. 欠損している標高値を補完する
3. 点群を領域ごとに分割する
4. 各領域の高さ変化を計算する
5. 高さ変化に応じて点群を削減する
6. 地面と地面以外を判定する
7. Delaunay三角形分割によってポリゴンを生成する
8. VRML形式で出力する

この中で、私が主に担当したのは 3〜5 のデータ削減処理です。

## Technical Points

このプロジェクトで工夫した点は、単に点数を減らすのではなく、地形や建物の特徴を残すために、領域ごとの高さ変化を用いて削減率を変えた点です。

都市データでは、平坦な道路や地面と、建物の側面・屋上・段差が同じ点群データ内に混在しています。  
そのため、全体に同じ削減処理をかけると、重要な構造が失われる可能性があります。

そこで、各タイルの `z_range` をもとに削減の粒度を変えることで、平坦な領域では大きく削減し、立体的な領域では細かく残すようにしました。

## Reflection

このプロジェクトを通して、3Dデータ処理では「データ量を減らすこと」と「形状の特徴を保つこと」のバランスが重要であることを学びました。

特に、同じ点群データでも、地面・建物・段差などの性質が場所によって異なるため、単純な一律処理ではなく、データの局所的な特徴に応じて処理を変える必要があると感じました。

また、VRMLとして出力することで、アルゴリズム上の削減結果が実際の見た目にどのように影響するかを確認でき、数値的な削減率だけでなく、視覚的な品質の重要性も理解できました。
